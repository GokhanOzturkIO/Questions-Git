1- Git yazılım geliştirmede kullandığımız bir versiyon kontrol sistemidir.

2- git pull remote'daki değişiklikleri lokalimizde çalıştığımız branche çeker ve merge eder. git fetch ise sadece değişiklikleri çeker.

	-Örnek "git pull": Arkadaşımız ile aynı branchte çalışıyoruz ve kendisi işlerini bitirdi ve pushladı. Bu değişiklikleri kendi lokalimize çekmemiz için git pull
	komutunu kullanırız.
	
	-Örnek "git fetch": Arkadaşımız yeni bir branch açtı ve pushladı. Fakat takıldığı bir yer var bizim bakmamız gerekiyor. Yeni açılan branchi lokalimize çekebilmemiz
	için öncelikle git fetch komutu ile remote'daki değişiklikleri getirmemiz gerekir.

3- 	3.1 Arkadaşımız commitlemiş ama pushlamamış olabilir. 
   	3.2 Kendi değişikliklerimizi commitlememiş olabiliriz. 
	3.3 conflict çıkmış olabilir. 
	3.4 pull etme yetkimiz olmayabilir.


4- origin kelimesi uzak sunucudaki repoyu temsil etmektedir.

5- HEAD üzerinde çalıştığımız güncel commit'i temsil eder.

6- Staging area veya index commit öncesi değişikliklerin geçici olarak beklediği alanı temsil eder.

7- Untracked file git tarafından takip edilmeyen dosyasyı takip eder. Örnek: .gitignore dosyası veya yeni eklediğimiz bir dosya.
 
8- Eğer .git dosyasını silersek projenin tüm sürüm geçmişini ve git ile ilgili ayaları siler. Kısacası git ile bağlantısı kesilir ve pushlamadığımız değişiklikleri kaybederiz.

9- git init sonrası otomatik olarak readme dosyası oluşturmak istiyorsak bu işlem için uygun bir bash script yazıp bu script içersinde git init komutunu çalıştırabliriz.
	 
	Örnek bash scirpt:
		#!/bin/bash

		# Git deposunu başlat
		git init

		# ReadMe.md dosyasını oluştur
		echo "# Proje Başlığı" > ReadMe.md

		# ReadMe.md dosyasını staging area'ya ekle
		git add ReadMe.md

		# İlk commit'i yap
		git commit -m "Initial commit with ReadMe.md"



10- Branch yapısı, ekiplerin birbirlerinin çalışmalarını etkilemeden paralel olarak çalışabilmesine olanak tanır ve 
    kodun ana dalına (master veya development) birleştirilmeden önce ayrı bir şekilde geliştirilip test edilmesine imkan verir.

	

11- Yeni bir branch oluşturmak için eğer git bash kullanıyorsak git branch <branch-ismi> komutunu çalıştırabiliriz. Git için bir arayüz programı kullanıyosak bunu
    uygulamanın sağladığı kısayollar ile de yapabiliriz.

12- Varolan bir branche git checkout <branch-ismi> komutu ile geçiş yapabiliriz.

13- Sadece belirli bir branchi clone etmek istiyorsak git clone -b <branchname> <remote-repo-url> komutu ile klonlayabiliriz.

14- Merge Conflict terimi bir dosyada aynı satırda değişikliklerin yapılması ve bu değişikliklerin merge edilmesi sırasında git tarafından çözülemediği
    durumu ifade eder. Manuel olarak çözmemiz beklenir.
